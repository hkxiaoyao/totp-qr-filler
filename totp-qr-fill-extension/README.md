# Ephemeral 2FA QR Filler

一个最小 Chrome MV3 插件：点击按钮后识别当前可见页面里的 `otpauth://totp/...` 二维码，计算 TOTP 验证码，并填入页面中的验证码输入框。

## 特性

- 使用 `chrome.storage.session` 在浏览器会话内临时保留 2FA secret
- 不上传二维码内容
- 不自动提交表单
- 使用 `activeTab` 临时权限
- 优先使用 Chrome 原生 `BarcodeDetector`，不支持时自动使用本地 `jsQR` 备用解码器
- 支持单个验证码输入框和常见的多格验证码输入框
- 关闭并重新打开弹窗后，会继续使用上次识别到的二维码参数刷新验证码
- 识别到新的二维码后，会覆盖旧参数

## 安装

1. 打开 `chrome://extensions`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本目录：`totp-qr-fill-extension`

## 使用

1. 打开包含 2FA 二维码和验证码输入框的页面
2. 确保二维码完整显示在当前可见区域
3. 如果页面输入框比较特殊，先点击验证码输入框让光标停在那里
4. 点击浏览器工具栏中的插件图标
5. 点击“识别并填充”

如果没有找到可填充的输入框，插件会显示当前验证码，并尽量复制到剪贴板，方便手动粘贴。

验证码参数保存在 Chrome 扩展的 session 存储中：关闭弹窗不会丢，识别到新的二维码会覆盖旧参数，关闭浏览器后会清除。

## 限制

- 当前只支持 TOTP，不支持 HOTP
- 只能识别当前可见区域里的二维码
- 不保存 secret，因此离开二维码页面后无法继续生成验证码
